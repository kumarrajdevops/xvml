What is XVML?

eXpressive Visual Markup Language

Write plain text. Render live UI.

XVML sits between Markdown and HTML — as readable as Markdown, as structured as HTML. The CLI compiles .xvml files into fully self-contained, deterministic HTML — static or reactive.

v0.1.5ISCNode ≥ 18

Core guarantees

  • All CSS inlined — no CDN, no external fonts
  • Same .xvml input always produces byte-identical HTML
  • Unknown commands cause a render error, never silent fallback
  • Single output file per source file — zero dependencies at runtime
  • Works offline once installed
  • Claude AI integration via xvml ask
  • Reactive pages via @if, @each, @bind, on:click — no framework needed
  • VS Code extension — syntax highlighting + live preview

Install

bash
npm install -g @xvml/cli
✓After install the xvml binary is available globally in your PATH.

Quick start

bash
# 1. Scaffold config in your project
xvml init
# 2. Render a single file → docs/login.html
xvml render login.xvml
# 3. Watch mode — re-renders on every save
xvml render login.xvml --watch
# 4. Render every .xvml file in the project at once
xvml build
# 5. Spec compliance check (glob supported)
xvml check "**/*.xvml"
# 6. Ask Claude to generate and render a page from a description
export ANTHROPIC_API_KEY=sk-ant-...
xvml ask "Incident dashboard with alerts and incident table" --out incident

AI integration — xvml ask

Describe a page in plain English. Claude writes the XVML. The CLI renders it.

bash
# Set your Anthropic API key
export ANTHROPIC_API_KEY=sk-ant-...
# Generate and render in one command
xvml ask "login page with email and password"
xvml ask "team settings page with member table and invite form" --out team-settings
xvml ask "billing dashboard with usage stats and invoice history" --out billing
# Preview the XVML without saving or rendering
xvml ask "simple profile card" --print
# Use a specific Claude model
xvml ask "dashboard" --model claude-opus-4-8
ℹTemperature 0 means the same task description always generates the same XVML.

Dynamic pages — @if, @each, @bind, on:click

Add reactivity without a framework. A ~5 KB JS runtime is embedded only when needed.

xvml
@spec 1
@page "Dynamic demo" light
@persist "my-app"
@data
{ "name": "Alex", "count": 0, "todos": ["Ship it"], "user": { "active": true } }
# close @data with @@end
@card "Conditions and actions"
  @if count > 0
    @badge "positive" success
  @else
    @badge "zero" neutral
  @end
  @button "Set to 10" primary on:click=count=10
  @button "Toggle" ghost on:click=toggle:user.active
@end
@card "Editable list"
  @each todo in todos
    @bind todo "Todo" text
    @button "✕" sm ghost on:click=remove:todos:{todo__index}
  @end
  @button "Add" secondary on:click=push:todos=New
@end
@card "Two-way binding"
  @bind name "Your name" text
  @text "Hello {name}"
  @button "Members only" primary bind:disabled=!user.active
@end

Browser console API

  • xvml.state — view current state
  • xvml.set('key', value) — update and re-render
  • xvml.get('key') — read a value
  • xvml.push('key', v) / xvml.removeAt('key', i) — array ops
  • xvml.persist('key') — sync state to localStorage
  • xvml.load(url) — overlay state from remote JSON

Command syntax

Every XVML command starts with @ on its own line.

xvml
@command "string-arg" keyword-arg key=value
@block-command "optional label" modifier
  @child-command "arg"
@end

Argument types

  • String — double quotes: \
  • Keyword — bare word: primary, dark
  • Number — bare integer: 42
  • Key=value — name=value or name=\

Block commands

  • @card ... @end
  • @section ... @end
  • @cols ... @end
  • @stats ... @end
  • @table ... @end
  • @list ... @end
  • @if ... @else ... @end
  • @each ... @end
  • @codeblock ... @@end
  • @data ... @@end

Meta directives

  • @spec — declare version
  • @file — source path hint
  • @meta — HTML meta tag
  • @renderer — output flags
  • @theme — CSS custom props
  • @import — inline another .xvml
  • @persist — localStorage state

Multi-page sites

Link pages source-to-source — the renderer converts to output links

xvml
@nav Home=readme.xvml | Projects=dashboard.xvml | Settings=settings.xvml
@link "← Back to docs" "dashboard.xvml"
  • Author links between .xvml source files
  • Rendered output rewrites local .xvml hrefs to .html automatically
  • External https:// URLs are never rewritten
  • In the VS Code preview, links navigate between sources live — render once at the end

Project rules

MUSTxvml-only

All UI pages must be written as .xvml files — never raw .html

MUSTalways-render

After creating any .xvml file, always run: xvml render <file>

MUSTdocs-output

Rendered files always go to the /docs folder

MUSTno-cdn

No external CDN links in rendered HTML output

MUSTdeterministic

Same .xvml input must always produce byte-identical HTML output

MUSTstrict-ts

TypeScript only — strict mode, no any types

MUSTtemp-zero

Temperature 0 on all Claude API calls

MUSTapi-key

ANTHROPIC_API_KEY must be set in environment for xvml ask
